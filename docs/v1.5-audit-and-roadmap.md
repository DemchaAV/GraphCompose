# GraphCompose v1.4.1 — Audit & v1.5+ Roadmap

> **Дата:** 2026-04-29
> **Версия:** v1.4.1 ("cinematic")
> **Цель документа:** свести в один план, что у движка уже есть, чего не хватает,
> и какими шагами довести библиотеку до состояния, при котором она **подталкивает**
> разработчика к правильным решениям, а не требует постоянно сверяться с docs.

---

## TL;DR

GraphCompose — это уже **зрелый Java-движок документов**: семантический DSL,
детерминистичный layout-проход, layer stacks, business-themes, page-backgrounds,
DOCX-export, 525 зелёных тестов, чистый canonical surface. Миграция с legacy
завершена на 90%+, allowlist'ы guard-тестов пусты — public API не «течёт» из engine.*

Слабые места — не в движке, а **по краям**:

1. **Главный пользовательский use-case (круг как контейнер с детьми, движущимися
   вместе с ним)** не выражается canonical DSL. `LayerStackNode` стэкает в общий
   bbox, но не клиппит детей по `path` shape’а. Это **флагманская фича v1.5**.
2. **Скрытые силы движка**: `Transform` (rotation, scale), 9-точечный `Anchor`,
   `Layer` z-index per-entity, opacity — реализованы внутри ECS, но не
   exposed через builder API. Часть из них ждёт всего «фасада».
3. **DSL ergonomics**: `LayerStackBuilder` имеет только `back/center` (а LayerAlign
   умеет 9 точек), `gap` vs `spacing` рассогласованы, нет fluent overloads
   (`addEllipse(60, color)`), late validation в `RowBuilder.build()`.
4. **Templates/examples/docs не догоняют v1.4**: ни один built-in template не
   использует `BusinessTheme`, `LayerStack`, `RichText` или `colSpan`. Новички видят
   v1.2-style PDF, не «cinematic» демо. `getting-started.md` ссылается на v1.2.
5. **CI/release tooling**: `visual-baselines/` пуст (PdfVisualRegression скипается),
   нет Maven Central deploy, нет API diff (semver guard), нет автогенерации
   changelog.

**Стратегия v1.5** = «Intuitive»: ввести **shape-as-container**, expose'ить
скрытые engine capabilities через builders, дотянуть `LayerStack`/`Row`/`Table`
до симметричного API, и **вытянуть templates на v1.4-уровень**, чтобы первое
впечатление от библиотеки соответствовало её реальной мощи.

## 2. Главные пять разрывов

### 2.1. Shape-as-container — флагманский запрос пользователя

> «Мы можем сделать круг, в него поставить какой-то элемент, который будет в
> границах круга и будет перемещаться вместе с ним.»

**Что есть сейчас**

| Узел / handler | Что делает | Чего нет |
|---|---|---|
| `EllipseNode` / `ShapeNode` | Рисует залитую/обведённую форму | Не принимает children |
| `EllipseBuilder` / `ShapeBuilder` | Параметры геометрии | Нет `addInside(...)` |
| `LayerStackNode` (v1.4) | Стекует children в общий bbox по `LayerAlign` | **Без клиппинга по path родителя** — child может вылезти за круг |
| `PdfShapeFragmentRenderHandler` | Рисует path круга/прямоугольника | Не вызывает `clipPath` перед детьми |
| `Transform`, `Layer` (per-entity) | ECS-компоненты есть | Не доходит до канонического DSL |

**В чём разрыв.**
LayerStack визуально кладёт `paragraph("M&A")` в центр bbox круга, но bbox прямоугольный.
Если текст шире `2r` — он торчит за окружность. Если круг переезжает страницу,
дочерний текст — тоже, но без гарантии «в тех же координатах относительно центра круга».
Это и есть «не движется вместе с ним полностью».

**Что нужно**

1. Новый узел `ShapeContainerNode` (или `clipChildren=true` + `addLayer(...)` на
   уже существующем `LayerStackNode`) — composite, у которого:
   - `outline` — `Shape` (rectangle / ellipse / rounded rect / polygon).
   - `children` с `Position(x, y) + Anchor` (см. ниже про Anchor).
   - `clipPolicy` — `CLIP_BOUNDS` (текущее), `CLIP_PATH` (новое), `OVERFLOW_VISIBLE`.
2. **Shape geometry pre-pass** в layout system: разрешить outline до того, как
   `PageBreaker` начнёт работу. Сейчас child-first нарушит: дети расчитаются до
   контейнера. Нужно ввести `ShapeClipPath`-компонент, который ставится на entity
   родителя в первом проходе, и `PageBreaker.paginationPolicy()` возвращает
   `SHAPE_ATOMIC` для таких контейнеров (не split'ить детей между страницами).
3. **PDF clip path**: в `PdfShapeFragmentRenderHandler` добавить вариант
   `drawWithClippedChildren(...)` — `saveGraphicsState() → moveTo+curveTo (Bezier
   approximation круга) → clip → drawChildren → restoreGraphicsState()`. Pattern
   уже частично есть для shape rendering, надо обернуть в push/pop.
4. **DOCX backend**: graceful degradation — вернуть `unsupported(reason)` или
   рендерить ребёнка без клиппинга. Документировать в parity matrix.
5. **Snapshot**: расширить `PlacedFragment` опциональным `clipPath` метаданным,
   чтобы layout-snapshot тесты ловили регрессии до пиксельной проверки.

**Целевой DSL** (примеры из user request):

```java
// Минимум: круг с одним центрированным ребёнком.
section.addCircle(60, brand, c -> c
        .center(child -> child.text("M&A")));

// Эллипс с children по 9 точкам.
section.addEllipse(e -> e
        .size(160, 100)
        .fillColor(brand)
        .clipChildren(true)
        .layer(badge, LayerAlign.TOP_RIGHT)
        .center(label));

// Отдельный builder для произвольной формы.
section.addShape(s -> s
        .roundedRect(180, 90, 12)
        .fillColor(slate)
        .accentLeft(gold, 4)
        .addInside(inner -> inner
                .padding(12)
                .addText("Featured proposal", theme.text().h2())
                .addRich(t -> t.plain("by ").bold("GraphCompose"))));
```

**Effort**: 25–35 SP (фаза B v1.5).

**Риски (раскрыты в §6):** child-first invariant, page-shift propagation,
LayoutSnapshot-консистентность, PDF graphics-state корректность.

### 2.2. Скрытые силы движка, не выведенные в DSL

ECS уже несёт компоненты, которые public API не использует. **Это самый дешёвый
источник новой выразительности — фасад уже почти есть.**

| Возможность | Где живёт | Exposure | Effort | Impact |
|---|---|---|---|---|
| **Transform (rotation, scaleX/Y)** | `engine/components/layout/Transform.java:19` | NO | M (15–20 SP) | HIGH — кардинально расширяет «cinematic» |
| **Anchor (HAnchor × VAnchor, 9 точек)** | `engine/components/layout/Anchor.java:27` | PARTIAL — компонент есть, builder без `.anchor(...)` | L (5 SP) | HIGH |
| **Layer/z-index per arbitrary entity** | `engine/components/layout/Layer.java:13` | PARTIAL — только в `LayerStackNode` | L (3–5 SP) | MEDIUM |
| **Per-element opacity** | пока только в `WatermarkConfig.opacity` | NO | M (12–18 SP) — нужно протолкнуть в payload type'ы и PDF graphics state | LOW–MEDIUM |
| **9-точечный alignment в LayerStack** | `LayerAlign` enum | PARTIAL — builder имеет только `back()`, `center()` | XS (1–2 SP) | HIGH (discoverability) |
| **Custom path / Bezier** | `LineFragmentPayload` есть; path — нет | NO | H (новый payload + handler) | LOW |

**Quick wins** (фаза A v1.5):

- `LayerStackBuilder.topLeft(node)`, `topCenter(node)`, …, `bottomRight(node)` —
  9 fluent shortcuts.
- `LayerStackBuilder.position(x, y, anchor)` — относительный сдвиг.
- `*Builder.zIndex(int)` mixin (доступен на `addParagraph/addShape/addEllipse/...`).
- `*Builder.rotate(double deg)`, `.scale(double sx, double sy)` — на любом content
  node, под капотом ставит `Transform` component.

### 2.3. DSL ergonomics & discoverability

| # | Friction | Где | Lift |
|---|---|---|---|
| 1 | `gap()` (Row) vs `spacing()` (Flow) — одно и то же | `RowBuilder`, `AbstractFlowBuilder` | Унифицировать в `spacing()`, оставить `gap()` как `@Deprecated` alias |
| 2 | `LayerStackBuilder` — только `back/center` | `LayerStackBuilder` | + 9 alignment shortcuts |
| 3 | Нет shortcut overload'ов | `addEllipse`, `addCircle`, `addShape`, `addImage` | Добавить `addEllipse(diameter, color)`, `addShape(w, h, color)`, `addImage(data, w, h)` |
| 4 | `DividerBuilder extends ShapeBuilder` → expose'ит `fillColor()` (бессмыслица для divider) | `DividerBuilder` | Выделить `LineLikeBuilder` mixin или переименовать общие методы; убрать наследование от `ShapeBuilder` |
| 5 | `addRich(RichText)` и `addRich(Consumer<RichText>)` — обе сигнатуры, неочевидно когда что | `AbstractFlowBuilder` | Оставить только Consumer; static factory `RichText.text(...)` для прямого pass |
| 6 | `RowBuilder.build()` валидирует «нет nested rows/tables» — ошибка поздняя | `RowBuilder.add(...)` | Перенести валидацию в `add(...)`, бросать `IllegalArgumentException` сразу |
| 7 | Нет nested list builder | `ListBuilder.addItem(String)` | `addItem(String label, Consumer<ListBuilder> nested)` |
| 8 | `TableBuilder` нет zebra / total-row helper'ов | `TableBuilder` | `.zebra(odd, even)`, `.totalRow(cells…)`, `.headerRow(cells…)` |
| 9 | `ColSpan` — есть; `RowSpan` — нет | `DocumentTableCell` / `TableLayoutSupport` | Реализовать row span (cell-grid preprocessing) |
| 10 | Нет `expandWidth()` / responsive size hint | `ShapeBuilder`, `ImageBuilder` | `.expandWidth()`, `.expandHeight()` (alias на «match parent inner box») |
| 11 | `RichText` — только static factory; нет `dsl.richText(rt -> ...)` | `DocumentDsl` | Добавить `DocumentDsl.richText(Consumer<RichText>)` |
| 12 | Нет `.advanced()` под-builder для редких настроек | повсеместно | Опц.: ввести `*.advanced(adv -> adv.stroke(...).borders(...))` для скрытия редко нужных параметров |

**Принцип**: каждое предложение — это **либо alias** (cheap), **либо overload**,
**либо валидация раньше**. Нет breaking changes, есть только новые сахарные пути.

## 3. DX-аксиомы: чему должна «подталкивать» библиотека

Эти семь правил — фильтр для всех новых API. Каждое предложение из §5 проверяется
ими.

1. **Discoverability через autocomplete.** Каждый builder должен иметь все
   осмысленные методы (включая 9 alignment'ов, не два). Если IDE показывает 8 точек,
   а есть 9 — это баг UX.
2. **Симметрия.** Если есть `accentLeft`, должны быть `accentRight/Top/Bottom`.
   Если есть `back/center`, должны быть и `topRight/bottomLeft/...`. Уже сейчас
   `accent*` симметричен — это хороший шаблон.
3. **Consistency naming.** `gap` и `spacing` — одно и то же; называется одинаково
   (`spacing` побеждает, потому что используется в большем числе мест).
4. **Fail-fast.** Ошибки структуры — в `add(...)`, не в `build(...)`. Чем раньше,
   тем дешевле починить.
5. **Theme inheritance.** Если задан `BusinessTheme` на documenta-уровне,
   `softPanel()` без аргументов берёт цвет из theme. Сейчас это требует явных
   `theme.palette().surfaceMuted()` calls.
6. **Progressive disclosure.** Простой случай — одной строкой
   (`addCircle(60, brand)`). Расширенный — через тот же путь
   (`addCircle(c -> c.diameter(60).fillColor(brand).clipChildren(true).addInside(...))`).
7. **Noun-style чтение.** API должен читаться как фраза. `section.addCircle(...).
   center(child -> ...)` — естественно. `section.layerStackOf(...).addLayer(...,
   LayerAlign.CENTER)` — нет.

## 4. Roadmap

### 4.1. v1.5 — «Intuitive» (4–6 недель)

Цель — закрыть **главный пользовательский запрос** (shape-as-container) и
довести API до состояния «пишется без открытого reference'а».

#### Phase A — Quick UX wins (1 неделя)

- [ ] `LayerStackBuilder`: 9 alignment shortcuts (`topLeft/topCenter/topRight/centerLeft/center/centerRight/bottomLeft/bottomCenter/bottomRight`) и `position(x, y, anchor)`.
- [ ] Унифицировать `gap()` → `spacing()`, оставить `gap()` `@Deprecated` alias.
- [ ] Convenience overload'ы: `addEllipse(diameter, color)`, `addCircle(diameter, color)`, `addShape(w, h, color)`, `addImage(data, w, h)`, `addLine(thickness, color)`.
- [ ] `accentStroke(color, width)`, `accentFill(color)` поверх существующих presets.
- [ ] `RowBuilder.add(...)` валидирует структуру немедленно.
- [ ] `DocumentDsl.richText(Consumer<RichText>)` entry point.
- [ ] `*Builder.expandWidth()`, `expandHeight()` для shapes/images.
- [ ] `ListBuilder.addItem(label, Consumer<ListBuilder>)` для nested lists.

Definition of done: CHANGELOG, javadoc, 1 test per shortcut, README не правится
(это сахар на существующих фичах).

#### Phase B — Shape-as-container + clipping (2–3 недели)

- [ ] Новый узел `ShapeContainerNode` или флаг `clipPolicy` на `LayerStackNode` /
  `ShapeNode`. Решить на ADR-уровне (см. §5.2).
- [ ] `EllipseBuilder.addInside(Consumer<...>)`, `ShapeBuilder.addInside(Consumer<...>)`,
  `addCircle(diameter, color, Consumer<...>)`.
- [ ] **Shape geometry pre-pass** в `LayoutCompiler`: outline резолвится
  до `PageBreaker`. Pin'ится `ShapeClipPath` component на entity.
- [ ] `PaginationPolicy.SHAPE_ATOMIC` (атомарный + child-first сохраняется внутри shape).
- [ ] `PdfShapeFragmentRenderHandler.drawWithClippedChildren(...)` — `saveGraphicsState
  → clipPath → drawChildren → restoreGraphicsState`.
- [ ] DOCX backend: graceful fallback (rendering без clip + warning).
- [ ] Layout snapshot: `PlacedFragment.clipPath` метаданные, snapshot diff умеет
  показывать clip path.
- [ ] Visual baseline: 3 фикстуры (circle-with-text, ellipse-with-overlay, rounded-rect-card).
- [ ] Architecture guard: `ShapeContainerArchTest` — child-first инвариант не нарушается.
- [ ] Recipe `docs/recipes/shape-as-container.md`.

#### Phase C — Transform & per-entity Layer (1–2 недели)

- [ ] `*Builder.rotate(double deg)`, `.scale(double sx, double sy)` — добавляет
  `Transform` component на entity.
- [ ] `*Builder.zIndex(int)` — добавляет `Layer` component.
- [ ] PDF render handlers пушат CTM (`saveGraphicsState → cm → draw → restore`)
  для transformed entities; `EntityRenderOrder` уважает явный `Layer`.
- [ ] Visual baseline: rotated banner, scaled badge.
- [ ] Recipe `docs/recipes/transforms.md`.

#### Phase D — Tables advanced (1–2 недели)

- [ ] `DocumentTableCell.rowSpan(n)` + cell-grid preprocessing в `TableLayoutSupport`.
- [ ] `TableBuilder.zebra(odd, even)`, `.totalRow(cells…)`, `.headerRow(cells…)`.
- [ ] **Repeated header при page break** (требует разбить `TableNode` атомарность
  по rows; per-row fragments вместо all-at-once). См. §6 риск 4.
- [ ] Visual baseline: long table с repeated header, zebra, total row.
- [ ] Recipe `docs/recipes/tables.md` дополнить.

#### Phase E — Templates & examples lift to v1.4 (параллельно с A–D)

- [ ] `InvoiceTemplateV2` — на `BusinessTheme.classic()`, `softPanel`, `accent*`,
  `colSpan` для total row, `RichText` для статусов.

## 5. Конкретные API-предложения

Все сигнатуры — pseudo-Java (без javadoc для краткости). Каждый блок — это
минимальный contract, который реализуется в фазе.

### 5.1. LayerStackBuilder: 9-точка + position

```java
public LayerStackBuilder topLeft(DocumentNode node);
public LayerStackBuilder topCenter(DocumentNode node);
public LayerStackBuilder topRight(DocumentNode node);
public LayerStackBuilder centerLeft(DocumentNode node);
public LayerStackBuilder center(DocumentNode node);
public LayerStackBuilder centerRight(DocumentNode node);
public LayerStackBuilder bottomLeft(DocumentNode node);
public LayerStackBuilder bottomCenter(DocumentNode node);
public LayerStackBuilder bottomRight(DocumentNode node);

public LayerStackBuilder position(DocumentNode node, double x, double y, LayerAlign anchor);
```

`back()` / `center()` остаются (back == bottomLeft semantics, deprecate'ить
не нужно, это удобный alias).

### 5.2. ShapeContainerNode (ADR-decision)

Два варианта:

**(a) Новый узел `ShapeContainerNode`** — отдельный node-type, который держит
`outline` (Shape) + `children` (List<DocumentNode> с Position+Anchor).

```java
public final class ShapeContainerNode implements DocumentNode {
    private final ShapeOutline outline;          // Rect | Ellipse | RoundedRect | Polygon
    private final List<PositionedChild> layers;
    private final ClipPolicy clipPolicy;         // CLIP_BOUNDS | CLIP_PATH | OVERFLOW
    private final NodeStyle style;
}
```

**(b) Расширить `LayerStackNode`** опцией `clipOutline` + `clipPolicy`, и
позволить `back/center/...` принимать `ShapeNode` как outline.

**Рекомендация — вариант (a).** Причины: (1) семантика «container» концептуально
другая, чем «stack»; (2) разные pagination policies (`SHAPE_ATOMIC` vs `ATOMIC`);
(3) проще учить — у пользователя в IDE появляется явный builder.

```java
section.addCircle(60, brand, c -> c.center(child -> child.text("M&A")));

section.addShape(s -> s
    .roundedRect(180, 90, 12)
    .fillColor(slate)
    .accentLeft(gold, 4)
    .clipPolicy(ClipPolicy.CLIP_PATH)
    .addInside(inner -> inner.padding(12)
        .addText("Featured", theme.text().h2())
        .addRich(t -> t.plain("by ").bold("GraphCompose"))));
```

### 5.3. Transform mixin

Добавить в `AbstractFlowBuilder` (и/или прямо на content-builder'ы):

```java
public T rotate(double degrees);
public T scale(double sx, double sy);
public T scale(double uniform);   // sx == sy
```

Под капотом: builder проставляет `Transform` component на entity. `LayoutSystem`
ничего не считает — Transform применяется на render time через CTM. Это сохраняет
child-first invariant (геометрия дочки не меняется, меняется только матрица отрисовки).

### 5.4. TableBuilder advanced

```java
public TableBuilder zebra(DocumentTableStyle odd, DocumentTableStyle even);
public TableBuilder totalRow(String... cells);            // последняя row, пред-стилизована
public TableBuilder headerRow(String... cells);           // alias к header(...)

```


## 6. Архитектурные риски shape-as-container

| Риск | Описание | Mitigation |
|---|---|---|
| **R1: Child-first invariant** | `PageBreaker` обрабатывает детей до контейнера. Если outline зависит от content size, дети расчитываются раньше outline'а | Ввести явную «shape geometry pre-pass» фазу до `PageBreaker`. Outline shape — функция от заданных размеров, не от content (для v1.5 поддерживаем только explicit `diameter` / `size`). Auto-fit отложить в v1.6. |
| **R2: Page-shift propagation** | Если ребёнок переезжает страницу, `ParentContainerUpdater` сдвигает координаты всем children. Для clipped child это значит «orphan вне clip rect» | `PaginationPolicy.SHAPE_ATOMIC`: shape container и все его дети — на одной странице или не печатаются (`oversized` policy с диагностикой) |
| **R3: LayoutSnapshot inconsistency** | snapshot'ы сейчас не содержат clipPath, регрессии не ловятся | Расширить `PlacedFragment` опциональным `clipPath` (детерминистичная сериализация Bezier path'а как `[(cmd, x, y)…]`). Update snapshot baselines |
| **R4: PDF graphics state leak** | Если clip path не закрыт `restoreGraphicsState`, последующий контент рендерится клипнутым | Reuse `saveGraphicsState/restoreGraphicsState` pattern из `PdfShapeFragmentRenderHandler:52`. Architecture-guard test «clip всегда зеркальный pop» |
| **R5: Repeated header пагинация** | Сейчас `TableNode.paginationPolicy()` = ATOMIC, поэтому breaks полностью переносят таблицу. Для repeated header нужно эмиттить per-row fragments | Это **отдельная фича Phase D**, не блокер для shape-as-container. Внедрять через `TableLayoutSupport.splitRows(...)` + per-row pagination |
| **R6: DOCX gap** | DOCX (Apache POI) не поддерживает arbitrary clip path | Документировать в `canonical-legacy-parity.md`: «DOCX рендерит ShapeContainer без clip; outline показывается как граница» |
